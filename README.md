## Presentation
- 要求是OO和Web二选一发表，所以我选择了自己主研的OO测试。
- 为了让没有机器学习基础的学生听懂我的Presentation，我在做了必要的解释至于，并不讲解复杂的模型(其实本文涉及的模型如今看来已经是很简单的了)。
- 限时4~6min，为了压缩时间且不至于一直念经，我半夜把自己关在会议室里疯狂计时诵读。

### Presentation思路
既然是做一个非学术交流的Presentation，那么重点应该放在文章描述的实验上。很多听众(本科生)并不熟悉机器学习的理论，演讲者无法在短时间(4min~6min)内讲明白机器学习晦涩的原理，倒不如认真分析论文的内容。

本文作者的思路是非常清晰的，论文表述也很清晰。我认为模型和评估指标可以一笔带过，应该强调以下几部分：
- 解答一些普遍的疑惑
- 提一下作者研究过程的重点内容
- 简单描述一下作者得出结论的过程
- 对作者研究的结论做一下归纳
- 思考和交流读此论文的收获
- 批判性思考论文所描述的研究过程(本文作者自己已有充分的表述)

### 复盘此次Presentations中的不足
- 选题问题
    - 很多人会倾向于选择看似简单的方向，其实未必是最合适的或是最容易出彩的
    - 很多人不是以主题选文章，而是看到一篇文章再反过来对主题，这样很有可能错过更好的文章
- 态度问题
    - 细节问题处理不当，表述漏洞百出，如刊物为ISSN、如机翻论文后自拟标题、如前后各种不搭、……
    - 无脑使用机器翻译，然后CV大法复制粘贴得到报告和PPT，这明显是态度问题
    - 不认真研读论文，浅尝辄止，所有的算法、方法都既不理解也不能讲明白
    - 明显没有事先对发言内容和时长做一个预估和调整，在能准备的情况下去拼临场发挥，态度不认真
- PPT问题
    - 事先没有对PPT的兼容性进行测试，马上发言却打不开PPT的案例有多起，更是没有准备PDF这样的备选方案
    - 普遍不知道应该选择排列计时播放
    - 很多人做的PPT只是花里胡哨，美观有余而实用性不足
    - PPT文字过多，有通读PPT之嫌
- 讲评问题
    - 语速过快或磕磕绊绊，熟练者多语速过快(与限时有关)、不熟练者磕磕绊绊
    - 心里对文章表述以及实验内容不熟悉，发言过于依赖于PPT文字或是手里的稿子
    - 不为听众考虑，只考虑自己喋喋不休的阐述(实际上这种人往往没读懂文章)
    - 讲述时一直在低头，不看听众们
